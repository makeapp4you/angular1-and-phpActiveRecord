Session

Session là thành phần không thể thiếu trong bất kỳ ứng dụng nào có sử dụng bộ nhớ tạm dành cho việc Đăng nhập, lưu trữ dữ liệu trên Server. Thư viện Session này sẽ giúp các bạn đơn giản hóa các thao tác với Session hơn bao giờ hết, với những phương thức đặt tên dễ hiểu là một lợi thế khi sử dụng.

Các phương thức cần biết

  • Session::get() : lấy dữ liệu từ Session
  • Session::insert() : thêm dữ liệu vào Session có sẵn
  • Session::update(): cập nhật Session, bỏ qua dữ liệu có sẵn
  • Session::destroy() : xóa Session

Lấy dữ liệu từ Session

	
		// Trả về dữ liệu nếu tồn tại dữ liệu và 'false' khi không có dữ liệu chỉ định
		Session:get('key')

Insert và Update vào Session

Các bạn cần lưu ý giữa Insert và Update. Session::insert($key, $dataArray) dùng để thêm tuần tự dữ liệu vào bên trong dữ liệu Array trước đó (Dữ liệu phải là một Array). Trong khi Session::update($key, $data) không cần phân biệt Array, String hay Object đều có thể lưu dữ liệu vào, tuy nhiên nếu có tồn tại dữ liệu trước đó thì sẽ bị ghi đè.

	
		// Cẩn trọng khi sử dụng Session::insert()
		Session::insert('user', [
		    'id' => $user['id'],
		    'email' => $user['email'],
		    'level' => $user['level']
		]);

		// Session::update() được khuyến khích sử dụng hơn
		Session::update('user', [
		    'id' => $user['id'],
		    'email' => $user['email'],
		    'level' => $user['level']
		]);

Hủy bỏ Session

	
		// Xóa bỏ toàn bộ dữ liệu trong Session
		Session::destroy();
