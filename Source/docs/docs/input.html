Input

Thư viện Input này được viết để nhận lại những dữ liệu được gửi lên từ người dùng đã được biến đổi phù hợp để chạy trên Web Application, cụ thể ở đây là AngularJS. Thư viện này sử dụng các phương thức có tên rất thân thiện và dễ xài hơn cách xài trực tiếp biến môi trường như $_GET hay $_POST.

Các phương thức cần biết

  • Input::isGet() : kiểm tra Request có phải GET hay không
  • Input::isExists() : kiểm tra biến có tồn tại phần tử hay thuộc tính trong Array hoặc Object
  • Input::get(): lấy dữ liệu chỉ định hoặc tất cả từ biến môi trường

Kiểm tra Request GET

	
		// 'true' nếu $_SERVER['REQUEST_METHOD'] == 'GET' và ngược lại là 'false'
		Input::isGet();

Kiểm tra biến

Đây là cách viết ngắn gọn hơn thay vì cú pháp isset($data['abc']) && !empty($data['abc']) và tương ứng với Object thì trong phương thức này sẽ đảm nhiệm luôn cho các bạn.

	
		// 'true' nếu $user có tồn tại phần tử hay thuộc tính 'email'
		// 'false' nếu không tìm thấy
		Input::isExists($user, 'email');

Lấy dữ liệu từ Request

Điều thú vị là trong Input::get('key') đã hỗ trợ các bạn kiểm tra giá trị có rỗng hay chưa. Nên vì thế các bạn sẽ nhận lại dữ liệu khi nó không rỗng và sẽ nhận giá trị false khi nó rỗng.

	
		//Lấy dữ liệu chỉ định tương tự như $_GET['user'] hay $_POST['user']
		Input::get('user'); 

		//Lấy toàn bộ dữ liệu được gửi
		Input::get();
